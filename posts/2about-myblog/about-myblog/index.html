🖥️我的增添修改#

  1. 更改根目录下的 astro.config.mjs

    • 更改站点url site：“https://fuxieyi.github.io/”,

  2. 注意,若不删除biome.yaml，在移动端显示会有bug

  3. 媒体资源都放在public文件夹下

  4. A-Readme-FXY文件夹存放文章模版和readme

  5. build.yml 也可以删除，原因如下：

  • build.yml 中的构建功能已经包含在 astro_auto_deploy.yml 中
  • Node.js 版本问题 build.yml 使用了 Node.js 22 和 23 版本 这些版本太新，可能不稳定 而 astro_auto_deploy.yml 使用的是更稳定的 Node.js 20
  • 对于个人博客项目，不需要这么复杂的构建检查 保留一个部署工作流就足够了

🍥可修改的基本信息#

  1. 在根目录下的 src 文件夹中，你可以找到 config.ts 我们来开始改写
  • title：你的博客主标题
  • subtitle：你的博客副标题。可选，在首页会显示为“主标题 - 副标题”
  • lang：博客显示语言。注释已经列出了一些常用的值，如：en, zh_CN, zh_TW, ja, ko
  • themeColor：hue值则是你的博客主题色，可以在你的博客右上角的画板图标确定喜欢的颜色再填写
  • banner：src：即banner图片，支持http/https URL
  • favicon：src：即网站图标，支持http/https URL
  • links：即友情链接，这些链接在导航栏上
  • avatar：即你的头像
  • name：即你的名字
  • bio：即个性签名，会显示在头像和名字下面
  • NavBarConfig 为导航栏设置的超链接。ProfileConfig 为你的用户的超链接
  • icon：你需要前往icones.js去搜索你想要的图标，比如QQ，则填写 fa6-brands ，
  • Fuwari默认支持这几种类型：fa6-brands, fa6-regular, fa6-solid, material-symbols。可以自己导入喜欢的图片相关配置内容。

    例如（在项目根目录下）
pnpm add -D @iconify-json/material-symbols

🚀相关环境依赖构建(在windwos下的开发调试)#

  1. 下载Git

  2. Node.js

  3. 通过Node.js在windwos环境下安装pnpm

    • npm install -g pnpm

  4. 在博客根目录下安装其所需依赖

    • pnpm install
    • pnpm add sharp

  5. 默认情况下，npm 和 pnpm 的全局依赖会安装在 C:\Users\你的用户名\AppData\Roaming\npm，我们可以修改到 D盘。

      1. npm config set prefix “D:\npm-global”
    • 验证npm config get prefix # 应该返回 D:\npm-global
      1. pnpm config set store-dir “D:.pnpm-store”
    • 验证pnpm config set store-dir “D:.pnpm-store”

  6. 在博客项目根目录下根据其要求安装相关依赖

    • pnpm install
    • pnpm add sharp、

    至此，完成部署落地

🚀创建两个仓库来管理博客（方便自动化部署）#

astro框架会生成dist文件夹，用来存放相关静态网页，所以我们可以将项目拆开两个部分（dist文件夹单独用来部署网页）

  1. 新建一私有仓库 MyBlog-Main-astro
  2. 新建用来挂载部署github.io的公共仓库 FuXieyi.github.io （仓库名要与github用户名一样哦）
  3. 设置TOKEN，用来自动部署
    • github设置—开发者设置—个人访问令牌—生成新令牌（经典）
    • 转至私有仓库MyBlog-Main-astro设置—机密与变量—操作—新建仓库机密（将TOKEN粘贴进去）命名为TOKEN
  4. 初始化本地仓库后推送至私有仓库MyBlog-Main-astro
    • 设置.gitignore，添加dist（这个后面自动化部署推动到公共仓库FuXieyi.github.io）
  5. .github\workflows下创建一个自动化部署脚本astro_auto_deploy.yaml
name: astro_auto_deploy

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Setup pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8
          run_install: false

      - name: Install dependencies
        run: pnpm install

      - name: Build site
        run: pnpm build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          PERSONAL_TOKEN: ${{ secrets.TOKEN }}
          EXTERNAL_REPOSITORY: FuXieyi/FuXieyi.github.io
          PUBLISH_BRANCH: main
          PUBLISH_DIR: ./dist
          force_orphan: true 
          commit_message: auto deploy
  1. 对于私有仓库的自动依赖更新，构建dependency-update.yaml
name: Dependency Update Check

on:
  schedule:
    - cron: '0 0 * * 0'  # 每周日运行
  workflow_dispatch:  # 允许手动触发

jobs:
  check-updates:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          
      - name: Install pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8
          
      - name: Check for outdated dependencies
        run: |
          pnpm install
          pnpm outdated
          
      - name: Check for security vulnerabilities
        run: pnpm audit
  • 在 package.json 中添加一些有用的脚本：
-     "preinstall": "npx only-allow pnpm"（删除这一行）
+     "preinstall": "npx only-allow pnpm",
+     "check-updates": "pnpm outdated",
+     "check-security": "pnpm audit",
+     "update-deps": "pnpm update --latest",
+     "update-interactive": "pnpm update -i"
  • 现在你可以使用以下命令来管理依赖：
  • 检查更新 pnpm check-updates # 检查过时的依赖 pnpm check-security # 检查安全漏洞
  • 更新依赖 pnpm update-deps # 更新所有依赖到最新版本 pnpm update-interactive # 交互式更新依赖
  • 自动检查 我添加的 GitHub Actions 工作流会每周自动检查依赖更新 你也可以在 GitHub 仓库的 Actions 标签页手动触发检查
  • 使用建议： 定期运行 pnpm check-updates 和 pnpm check-security 检查更新 使用 pnpm update-interactive 进行交互式更新，这样可以更好地控制更新过程 更新后记得测试项目，确保一切正常运行
  • 这样的设置虽然没有 Dependabot 那么自动化，但对于私有仓库来说是一个很好的替代方案。你可以根据需要调整 GitHub Actions 的运行频率（目前设置为每周日）。

🧞 指令#

CommandAction
pnpm install 并 pnpm add sharp安装依赖
pnpm dev在 localhost:4321 启动本地开发服务器
pnpm build构建网站至 ./dist/
pnpm preview本地预览已构建的网站
pnpm new-post <filename>创建新文章
pnpm astro ...执行 astro add, astro check 等指令
pnpm astro --help显示 Astro CLI 帮助
